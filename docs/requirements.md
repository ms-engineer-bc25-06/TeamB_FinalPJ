> 5 歳の子どもを育てるワーキングマザー、みさきさん。

> 「またこころんと話したい！」と、子ども自身が楽しみにする日課にもなっている。

---

#### 🧾 データ管理ルール

- **保存期間**：感情記録データは原則 1 年間保存。1 年経過した記録は自動削除またはアーカイブ。
- **整合性**：
  - `user_id` は必須。退会時はユーザー情報削除とともに感情記録も削除。
  - 感情タグは感情マスターに存在するもののみ登録可。
- **重複制御**：
  - 同一日に複数回記録は可能とする（ただし後の集計で日単位の上限をつける可能性あり）

---

#### 📊 データ利用想定

- 感情記録の履歴表示（カレンダー or リスト）
- スタンプの累計数に応じたバッジ付与・アニメーション表示
- 将来的には感情の傾向グラフ（週・月別）への展開も視野に入れる

### 9. 外部連携・API 要件

| 外部サービス  | 概要                           | 方法         | 形式 |
| ------------- | ------------------------------ | ------------ | ---- |
| Whisper API   | 音声入力のテキスト変換         | REST API     | JSON |
| Firebase Auth | ユーザー認証機能               | Firebase SDK | -    |
| Firestore     | NoSQL データベース             | Firebase SDK | -    |
| Cloud Storage | 音声ファイルの保存             | Firebase SDK | -    |
| Vercel        | アプリのデプロイ／ホスティング | GitHub 連携  | -    |

> 🔍 補足：Whisper API は音声データを Base64 または Blob で送信、JSON でテキストを受け取る構成を予定

### 10. 制約事項・前提条件

- ユーザー登録は保護者のみ。子どもは保護者のアカウント内で記録する想定
- 一部機能はスマートフォン・タブレット向けに最適化（PC では UI 調整が必要）

### 11. リスクと対応方針

<!-- TODO: 何を書くか確認して、記載 -->

| リスク内容                                   | 対応方針                                                                 |
| -------------------------------------------- | ------------------------------------------------------------------------ |
| 音声認識の精度が低い場合がある               | テキスト入力による代替手段を提供／再録機能をつける                       |
| 子どもが毎日スマホを触ることによる依存       | 利用時間を制限するガイドライン提示／操作補助を親に促す                   |
| Whisper API が有料化・上限超過になる可能性   | 月間使用量を制限し、無料枠内での利用にとどめる仕組みを検討               |
| 感情カードや履歴データに不整合が起きる可能性 | バックアップ保存＋バリデーションチェックを実装                           |
| フロント・バックエンドの同期不具合           | ローディング状態の明示／再送信処理を備える                               |
| アクセス集中時のパフォーマンス低下           | Vercel CDN + Firebase バックエンドで負荷分散／スケーラブル設計を心がける |
